# UOB Turbine 7 Portal - Multi-Module Project

This is a multi-module Maven project refactored from the original single-module structure.

## Project Structure

```
uob-turbine7-portal-mm/
├── pom.xml                          # Parent POM
├── torque-orm/                      # Module 1: Torque ORM
│   ├── pom.xml
│   └── src/
│       ├── main/
│       │   ├── java/com/uob/om/     # OM classes (GtpUser, GtpRole, etc.)
│       │   └── torque-schema/       # Torque schema definitions
│       └── test/                    # Unit tests
├── turbine-model-controller/        # Module 2: Turbine MVC
│   ├── pom.xml
│   └── src/main/java/
│       ├── modules/                 # Turbine screens and actions
│       ├── flux/                    # Flux framework components
│       ├── services/                # Turbine services
│       └── wrapper/                 # Wrapper classes
├── spring-rest-api/                 # Module 3: Spring REST API
│   ├── pom.xml
│   └── src/main/java/com/uob/
│       ├── controller/             # REST controllers
│       ├── dto/                     # Data Transfer Objects
│       ├── config/                  # Spring configuration
│       └── service/                # Service layer (TurbineSecurityService)
└── webapp/                          # Module 4: Web Application
    ├── pom.xml
    └── src/main/
        ├── webapp/                  # Web resources (WEB-INF, templates, etc.)
        └── data/                    # SQL data files
```

## Modules

### 1. torque-orm
- **Purpose**: Database access layer using Torque ORM
- **Contains**:
  - Torque schema definitions (`gtp-security-schema.xml`, etc.)
  - Generated and custom OM classes (`GtpUser`, `GtpRole`, `GtpPermission`, etc.)
  - Base classes generated by Torque
- **Dependencies**: Torque Runtime, Fulcrum Security API

### 2. turbine-model-controller
- **Purpose**: Turbine MVC framework components
- **Contains**:
  - Turbine screens (Velocity templates handlers)
  - Turbine actions (request handlers)
  - Flux framework components
  - Turbine services and utilities
- **Dependencies**: torque-orm, Turbine Core, Fulcrum

### 3. spring-rest-api
- **Purpose**: Spring Framework REST API integration
- **Contains**:
  - REST controllers (`UserRestController`, `GroupRestController`, etc.)
  - DTOs for API responses
  - Spring configuration (`SpringConfig`, `SpringWebAppInitializer`)
  - Service layer bridging Turbine and Spring
- **Dependencies**: torque-orm, turbine-model-controller, Spring Framework

### 4. webapp
- **Purpose**: Web application packaging with deployment profiles
- **Contains**:
  - Web resources (WEB-INF, templates, static resources)
  - SQL data files
  - Docker resources
- **Profiles**:
  - `tomcat` (default): Generates WAR for Apache Tomcat
  - `jboss8`: Generates WAR for JBoss 8 (WildFly)

## Building the Project

### Build all modules:
```bash
mvn clean install -DskipTests
```

### Build for Tomcat (default):
```bash
mvn clean package -DskipTests -Ptomcat
```
Output: `webapp/target/uob-t7-portal-tomcat.war`

### Build for JBoss 8:
```bash
mvn clean package -DskipTests -Pjboss8
```
Output: `webapp/target/uob-t7-portal-jboss8.war`

### Build specific module:
```bash
cd torque-orm
mvn clean install
```

## Deployment

### Tomcat Deployment
1. Build with `-Ptomcat` profile (or use default)
2. Deploy `webapp/target/uob-t7-portal-tomcat.war` to Tomcat's `webapps/` directory
3. Access at: `http://localhost:8080/uob-t7-portal-tomcat/`

### JBoss 8 (WildFly) Deployment
1. Build with `-Pjboss8` profile
2. Deploy `webapp/target/uob-t7-portal-jboss8.war` to JBoss deployment directory
3. Access at: `http://localhost:8080/uob-t7-portal-jboss8/`

## REST API Endpoints

After deployment, REST APIs are available at:
- `http://localhost:8080/{context}/api/users`
- `http://localhost:8080/{context}/api/groups`
- `http://localhost:8080/{context}/api/roles`
- `http://localhost:8080/{context}/api/permissions`

## Notes

- The parent POM manages common dependencies and versions
- Each module can be built independently
- Module dependencies are resolved automatically by Maven
- Profiles allow different WAR configurations for different application servers
